Teleport — cross-folder & cross-Mac conversations
Teleport pulls a recent Claude Code conversation from another folder — or another Mac on your LAN — into your current session. Claude Code ties each conversation to the folder it ran in, so you lose the thread when you start Claude in the wrong project folder, or pick a project up on a different computer. Teleport carries it over for continuity.

You use it from chat — the /restwalker:teleport skill or the teleport MCP tools. Just ask, e.g. “what was I doing in myproject” or “pull the conversation from my other Mac.” There's nothing to run on this page; it's read-only and never modifies the source.

The settings below only configure teleport: how far back it pulls by default, an optional shared token for authenticated access, and whether this Mac lets other Macs pull its conversations.
Default window
How far back teleport pulls by default (e.g. 1h, 6h, 24h)
Pairing token (optional)
If set, both Macs must share it (HMAC-signed, never sent over the wire). Leave blank to allow any Mac on the LAN — trusted networks only.
Advertise on LAN
Let other Macs discover AND pull conversations from this Mac. Trusted networks only; needs the daemon on 0.0.0.0 and a daemon restart to take effect.
